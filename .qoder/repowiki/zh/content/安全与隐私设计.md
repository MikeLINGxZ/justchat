## 数据本地化存储机制

本项目采用本地SQLite数据库存储所有用户数据，包括聊天记录、模型配置等敏感信息。所有数据均保存在用户本地设备中，不会上传至任何远程服务器，从根本上保障用户隐私安全。

数据持久化通过GORM框架实现，使用SQLite作为底层数据库驱动。核心数据模型包括聊天会话（Chat）、消息记录（Message）、模型配置（Model）和供应商信息（Provider），均通过结构化方式存储于本地数据库中。

## 数据库路径与环境变量控制

数据库文件的默认存储路径由系统环境变量 `LEMONTEA_DB_PATH` 控制。当该环境变量存在时，系统将优先使用其指定的路径作为数据库文件位置。

若未设置环境变量，系统将自动在用户主目录下创建 `lemontea/data.db` 作为默认数据库文件路径。程序启动时会自动创建必要的目录结构，并确保路径可写。

```go
func getDbPath() (string, error) {
	if os.Getenv("LEMONTEA_DB_PATH") != "" {
		return os.Getenv("LEMONTEA_DB_PATH"), nil
	}
	// 获取用户 home 目录
	homeDir, err := os.UserHomeDir()
	if err != nil {
		return "", err
	}

	// 构建数据库路径
	dbPath := filepath.Join(homeDir, "lemontea", "data.db")

	// 确保目录存在
	dbDir := filepath.Dir(dbPath)
	if err := os.MkdirAll(dbDir, 0755); err != nil {
		return "", err
	}
	return dbPath, nil
}
```

## 数据库文件访问权限建议

为增强数据库文件的安全性，建议对数据库文件设置适当的文件系统权限。推荐配置如下：

- 文件权限应设置为 `600`（即 `-rw-------`），仅允许文件所有者读写
- 存储目录权限应设置为 `700`（即 `drwx------`），限制其他用户访问
- 在多用户环境中，确保数据库文件位于当前用户的私有目录中

当前代码中创建目录时使用 `0755` 权限，建议在生产环境中调整为更严格的 `0700` 权限以提高安全性。

## 前端加密辅助函数分析

前端 `crypto.ts` 文件提供了基础的密码处理工具函数，主要用于用户密码的哈希转换和强度验证：

- `hashPassword(password)`：使用MD5算法将原始密码转换为哈希值
- `validatePassword(password)`：验证密码强度，要求至少6位且包含字母和数字

这些函数可用于前端用户认证场景中的密码预处理，但需注意MD5算法已不推荐用于安全敏感场景，建议未来升级为更安全的哈希算法（如bcrypt或PBKDF2）。

```typescript
export const hashPassword = (password: string): string => {
  return MD5(password).toString();
};

export const validatePassword = (password: string): {
  isValid: boolean;
  message?: string;
} => {
  if (password.length < 6) {
    return { isValid: false, message: '密码至少6位字符' };
  }
  
  if (!/^(?=.*[a-zA-Z])(?=.*\d)/.test(password)) {
    return { isValid: false, message: '密码必须包含字母和数字' };
  }
  
  return { isValid: true };
};
```

## 无网络上报机制的设计原则

本项目严格遵循"零数据上报"的设计原则，所有数据处理均在本地完成，不包含任何形式的网络遥测或数据收集机制。这一设计确保了：

- 所有聊天记录完全保留在用户设备本地
- 模型配置和使用习惯不会被追踪
- 无任何形式的用户行为分析或数据回传
- 不依赖任何外部API进行核心功能实现

通过将数据存储完全本地化，系统避免了网络传输过程中的数据泄露风险，为用户提供了最高级别的隐私保护。

## 未来安全增强建议

为进一步提升系统的安全性和隐私保护能力，建议考虑以下增强措施：

1. **数据库加密**：引入SQLCipher等技术对SQLite数据库进行透明加密，即使数据库文件被窃取也无法直接读取内容
2. **生物识别解锁**：集成操作系统级生物识别（如指纹、面部识别）功能，在应用启动时进行身份验证
3. **加密算法升级**：将MD5替换为更安全的密码哈希算法，如Argon2或bcrypt
4. **自动锁定机制**：设置闲置超时自动锁定功能，保护临时离开时的会话安全
5. **权限最小化**：进一步收紧文件和目录的访问权限，遵循最小权限原则

这些改进将显著提升系统的整体安全水平，为用户提供更全面的隐私保护。
