## 构建失败（缺少 Wails CLI）

### 错误现象
执行 `wails3 build` 或 `wails3 dev` 时提示命令未找到，或构建过程报错中断。

### 可能原因
- 未安装 Wails CLI 工具
- Wails CLI 未正确添加到系统 PATH 环境变量
- 使用了错误的命令（如 `wails` 而非 `wails3`）

### 解决方案
1. 安装 Wails CLI：
   ```bash
   go install github.com/wailsapp/wails/v3/cmd/wails3@latest
   ```
2. 确保 `$GOPATH/bin` 已加入系统 `PATH`。
3. 验证安装：
   ```bash
   wails3 version
   ```
4. 使用项目 Taskfile 运行开发模式：
   ```bash
   task dev
   ```

## 绑定类型未更新（需重新 generate）

### 错误现象
前端调用后端方法时报错，如 `Service.GetModels is not a function`，或返回数据结构与预期不符。

### 可能原因
- 后端 Go 结构体或服务方法变更后，未重新生成前端绑定代码
- 前端使用的绑定文件（`bindings/` 目录下）未同步更新

### 解决方案
1. 重新生成绑定代码：
   ```bash
   wails3 generate
   ```
2. 检查 `frontend/bindings/` 目录下的类型定义是否更新，例如：
   - `frontend/bindings/gitlab.linhf.cn/project/lemontea/lemon_tea_desktop/backend/models/view_models/models.ts`
   - `frontend/bindings/gitlab.linhf.cn/project/lemontea/lemon_tea_desktop/backend/service/service.ts`
3. 清理并重启开发服务器：
   ```bash
   task dev
   ```

## 数据库连接错误（路径权限问题）

### 错误现象
应用启动时报错 `Failed to connect to database`，或无法读写数据。

### 可能原因
- 数据库文件路径无写入权限
- 自定义数据库路径环境变量 `LEMONTEA_DB_PATH` 设置错误
- 目标目录不存在且无法创建

### 解决方案
1. 检查 `LEMONTEA_DB_PATH` 环境变量是否设置正确：
   ```bash
   export LEMONTEA_DB_PATH="/Users/yourname/lemontea/data.db"
   ```
2. 若未设置环境变量，程序将默认在 `~/lemontea/data.db` 创建数据库，请确保该路径可写。
3. 手动创建目录并赋予权限：
   ```bash
   mkdir -p ~/lemontea
   chmod 755 ~/lemontea
   ```
4. 查看日志确认错误详情：
   ```bash
   # 启动时观察控制台输出
   task dev
   ```

## 窗口无法显示（前端路由或 Wails 配置错误）

### 错误现象
主窗口或设置窗口无法打开，页面空白或白屏。

### 可能原因
- Wails 窗口配置中 `URL` 路由错误（如 `/home` 或 `/settings` 不存在）
- 前端路由未正确配置对应页面
- 窗口创建逻辑出错

### 解决方案
1. 检查主窗口配置：
   - 确保 `main.go` 中 `URL: "/home"` 对应的页面存在
   - 确认 `frontend/src/pages/home/index.tsx` 已正确导出组件
2. 检查设置窗口逻辑：
   - `OpenSettingsWindow` 方法中 `URL: "/settings"` 需有对应页面
   - 确保 `frontend/src/pages/settings/index.tsx` 存在
3. 使用 `wails doctor` 检查环境与配置：
   ```bash
   wails3 doctor
   ```
4. 查看前端控制台是否有 404 路由错误或资源加载失败。

## 日志查看与调试建议

### 查看 Go 后端日志
- 所有日志通过 `backend/pkg/logger` 包输出
- 支持 `Info`、`Warn`、`Error`、`Debug`、`Panic` 等级别
- 错误日志会自动打印堆栈信息
- 使用方式示例：
  ```go
  logger.Errorf("Failed to connect to database: %v", err)
  logger.Info("Application started")
  ```

### 查看前端控制台错误
- 前端使用 `errorHandler.ts` 统一处理错误
- 支持提取 API 错误码并转换为用户友好提示
- 可通过 `extractErrorMessage(error)` 获取可读消息
- 网络错误、认证失败等均有映射提示

### 调试建议
1. 运行 `wails3 doctor` 检查环境健康状态
2. 开发模式下使用 `task dev` 启动，实时查看日志
3. 后端错误优先查看控制台输出的 `ERROR` 日志
4. 前端错误打开浏览器开发者工具，查看 Console 和 Network 面板
5. 对于数据绑定问题，确认 `wails3 generate` 已执行
