## 开发模式与热重载

项目在开发阶段通过 `wails3 dev` 命令启动，支持热重载调试。该命令读取 `build/config.yml` 中的配置，并监听前端和后端代码的变化，自动重新编译并刷新应用界面，极大提升开发效率。

`Taskfile.yml` 中定义了 `dev` 任务，调用 `wails3 dev` 并指定配置文件路径和端口，确保前后端协同工作。开发模式下，前端通过 Vite 的开发服务器运行，后端 Go 代码变更后自动重建并重启应用。

## 生产构建流程

生产环境构建通过 `wails3 build` 命令生成原生二进制文件。该流程分为多个阶段：前端构建、Go 代码编译、资源嵌入和最终打包。

`Taskfile.yml` 中的 `build` 任务根据当前操作系统（`{{OS}}`）调用对应平台的构建任务（如 `darwin:build`、`linux:build`、`windows:build`），确保跨平台兼容性。构建过程中会自动执行 `go mod tidy` 清理依赖，并生成前端绑定代码。

## 跨平台构建任务解析

项目使用 `Taskfile.yml` 实现跨平台构建任务管理，支持 macOS、Windows 和 Linux 三大平台。

### 环境变量设置
各平台任务通过 `env` 字段设置特定环境变量：
- **macOS**: 设置 `GOOS=darwin`、`CGO_ENABLED=1` 及最低部署版本 `MACOSX_DEPLOYMENT_TARGET=10.15`
- **Linux**: 设置 `GOOS=linux`、`CGO_ENABLED=1`
- **Windows**: 设置 `GOOS=windows`、`CGO_ENABLED=0`（静态链接）

### 资源嵌入
通过 `//go:embed all:frontend/dist` 指令将前端构建产物嵌入二进制文件，确保应用运行时无需外部资源目录。

### 图标打包
使用 `wails3 generate icons` 命令将 `appicon.png` 转换为各平台图标格式：
- macOS: 生成 `.icns` 文件
- Windows: 生成 `.ico` 文件

## 前端构建配置

前端使用 Vite 构建，配置文件 `vite.config.ts` 定义了开发服务器和路径别名。关键配置如下：
- `server.host` 设置为 `0.0.0.0`，允许外部访问
- `resolve.alias` 定义 `@` 指向 `src` 目录，`@bindings` 指向 `bindings` 目录，便于模块导入

`package.json` 中定义了构建脚本：
- `build:dev`: 开发模式构建，不压缩代码
- `build`: 生产模式构建，启用压缩

`Taskfile.yml` 中的 `build:frontend` 任务根据 `PRODUCTION` 环境变量选择构建命令，确保开发与生产环境分离。

## 资源嵌入与图标打包

项目通过 `wails3 generate icons` 命令将 `build/appicon.png` 转换为各平台图标格式，并在构建时自动嵌入。

- **macOS**: 图标文件为 `build/darwin/icons.icns`，打包时复制到 `.app` 包的 `Resources` 目录
- **Windows**: 图标文件为 `build/windows/icon.ico`，通过 `wails3 generate syso` 生成 `.syso` 资源文件，链接到可执行文件
- **Linux**: 使用 `appicon.png` 生成 `.desktop` 文件和 AppImage 包图标

## 最终打包产物与运行方式

### 打包产物位置
所有构建产物均输出至 `bin/` 目录：
- **macOS**: `bin/lemon_tea_desktop`（可执行文件）或 `bin/lemon_tea_desktop.app`（应用包）
- **Linux**: `bin/lemon_tea_desktop`（可执行文件），可选生成 AppImage、deb、rpm 包
- **Windows**: `bin/lemon_tea_desktop.exe`（可执行文件），可选生成 NSIS 安装包或 MSIX 包

### 运行方式
- **开发模式**: 执行 `task dev` 启动热重载开发环境
- **生产模式**: 直接运行 `bin/lemon_tea_desktop`（或 `.exe`）启动应用
- **macOS 应用包**: 双击 `.app` 文件运行
- **Windows 安装包**: 运行生成的安装程序部署应用

## CI/CD 集成建议

建议使用 GitHub Actions 实现自动化发布流程，配置多平台构建流水线：

1. **触发条件**: `push` 到 `main` 分支或创建发布标签
2. **构建矩阵**: 使用 `matrix` 策略并行构建 macOS、Windows、Linux 版本
3. **缓存优化**: 缓存 `node_modules` 和 Go 模块，加速构建
4. **产物上传**: 使用 `actions/upload-artifact` 上传各平台构建产物
5. **自动发布**: 结合 `softprops/action-gh-release` 在创建 Release 时自动上传所有平台包

示例工作流应包含：
- 安装 Node.js 和 Go 环境
- 安装 `wails3` CLI 工具
- 执行 `task package` 生成各平台发布包
- 签名 macOS 和 Windows 应用（如需）
