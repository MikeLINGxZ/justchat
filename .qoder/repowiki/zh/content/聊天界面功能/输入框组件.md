## 简介
输入框组件（ChatInput）是聊天界面的核心交互元素，负责处理用户消息输入、模型切换、文件上传等关键功能。该组件采用React函数式组件设计，结合TypeScript类型系统，实现了高度可复用和可维护的UI组件。组件通过props接收外部状态和回调函数，实现了与父组件的松耦合通信。

## 用户输入处理机制
组件通过受控组件模式管理文本输入状态，使用`useState`钩子维护`inputValue`状态。`handleInputChange`回调函数在每次输入变化时更新状态，并触发高度调整。

```mermaid
flowchart TD
Start([输入变化]) --> HandleChange["handleInputChange事件触发"]
HandleChange --> UpdateState["更新inputValue状态"]
UpdateState --> RequestFrame["requestAnimationFrame"]
RequestFrame --> AdjustHeight["adjustTextareaHeight"]
AdjustHeight --> End([完成])
```

## 中文输入法兼容性处理
为解决中文输入法在Composition状态下的回车冲突问题，组件实现了完整的composition事件处理机制。通过`isComposing`状态标记中文输入过程，在此期间禁用回车发送功能。

```mermaid
stateDiagram-v2
[*] --> Normal
Normal --> Composing : compositionstart
Composing --> Normal : compositionend
Normal --> Send : Enter(非Shift)
Composing --> Composing : Enter(忽略)
```

## 回车与换行逻辑分离
组件精确区分了回车发送和换行操作，通过检测`Shift`键状态实现逻辑分离。普通回车触发消息发送，`Shift+Enter`则插入换行符，满足不同输入场景需求。

```mermaid
flowchart TD
KeyDown["键盘事件"] --> CheckComposing{"isComposing?"}
CheckComposing --> |是| Ignore["忽略回车"]
CheckComposing --> |否| CheckEnter{"key === 'Enter'?"}
CheckEnter --> |否| Pass["传递事件"]
CheckEnter --> |是| CheckShift{"shiftKey?"}
CheckShift --> |是| InsertLine["插入换行符"]
CheckShift --> |否| Prevent["preventDefault"]
Prevent --> Send["触发handleSend"]
```

## 模型选择下拉菜单实现
模型选择功能通过`showModelMenu`状态控制下拉菜单的显示隐藏。菜单项动态渲染`availableModels`数组，每个选项点击后通过`onModelChange`回调通知父组件模型变更。

```mermaid

Component->>Component : toggle showModelMenu
User->>Component : 选择模型
Component->>Component : handleModelSelect
Component->>Parent : onModelChange(model)
Component->>Component : hide menu
```

## 文件上传功能交互


## 响应式高度调整机制
组件实现了智能的文本域高度自适应功能，通过`adjustTextareaHeight`函数动态调整`textarea`高度。使用`requestAnimationFrame`优化性能，避免频繁的DOM重排。

```mermaid
flowchart TD
InputChange["输入变化"] --> RAF["requestAnimationFrame"]
RAF --> GetScroll["获取scrollHeight"]
GetScroll --> CheckHeight{"高度变化?"}
CheckHeight --> |否| End1["结束"]
CheckHeight --> |是| SetHeight["设置新高度"]
SetHeight --> CheckMax{"超过maxHeight?"}
CheckMax --> |否| SetAuto["height:auto; overflow:hidden"]
CheckMax --> |是| SetMax["height:maxHeight; overflow:auto"]
SetAuto --> UpdateRef["更新lastHeightRef"]
SetMax --> UpdateRef
UpdateRef --> End2["完成"]
```

## 发送按钮状态管理
发送按钮的状态由`inputValue`内容和`isGenerating`状态共同决定。当输入为空时按钮禁用，正在生成消息时显示停止按钮，实现了清晰的用户反馈。

```mermaid
stateDiagram-v2
[*] --> Idle
Idle --> Disabled : inputValue为空
Idle --> Enabled : inputValue非空
Enabled --> Generating : isGenerating=true
Generating --> Stopping : 显示停止按钮
Stopping --> Idle : 停止生成
Enabled --> Idle : 发送消息
```

## 与父组件通信机制
组件通过一系列回调函数与父组件通信，形成了清晰的单向数据流。`onSendMessage`回调传递用户输入内容，`onModelChange`同步模型选择状态，实现了父子组件间的有效协作。

```mermaid
sequenceDiagram
participant Child as ChatInput
participant Parent as Chat组件
Child->>Child : 用户输入消息
Child->>Child : 点击发送
Child->>Parent : onSendMessage(消息内容)
Parent->>Parent : 处理消息发送
Parent->>Child : 更新isGenerating状态
Child->>Child : 显示停止按钮
```
