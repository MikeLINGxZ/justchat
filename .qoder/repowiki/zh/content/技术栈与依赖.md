## 技术栈概览

本项目采用现代化全栈技术组合，以前端React 19 + TypeScript + Vite为核心，后端以Go语言 + Wails v3为驱动，结合GORM与SQLite实现本地数据持久化。整体架构兼顾开发效率、运行性能与跨平台能力，适用于高性能AI客户端应用的快速迭代与部署。

## 前端技术栈详解

### React 19（函数组件与Hooks）
项目前端采用React 19版本，充分利用函数式组件与Hooks机制实现组件逻辑复用与状态管理。通过`useState`、`useEffect`等内置Hook以及自定义Hook（如`useModels`、`useViewportHeight`）提升代码可维护性与可测试性。

### TypeScript（类型安全）
TypeScript作为核心开发语言，提供静态类型检查，增强代码健壮性。项目通过`tsconfig.json`系列配置文件统一编译选项，确保类型系统在开发阶段捕获潜在错误。

### Vite（构建工具）
Vite作为前端构建工具，提供极速的开发服务器启动与热更新能力。通过`vite.config.ts`配置模块解析、CSS处理及开发服务器端口，显著提升开发体验。

### Ant Design（UI组件库）
采用Ant Design 5.x版本构建统一UI风格，结合`@ant-design/v5-patch-for-react-19`补丁确保与React 19兼容。通过Sass变量定制主题，实现视觉一致性。

### Zustand（轻量状态管理）
Zustand用于管理全局状态（如认证状态`authStore`），相比Redux更轻量且无需样板代码，适合中小型应用的状态管理需求。

## 后端技术栈详解

### Go语言（高性能服务逻辑）
Go作为后端主语言，利用其高并发、低延迟特性处理AI请求与本地业务逻辑。项目入口为`main.go`，通过Wails框架桥接前后端。

### Wails v3（桌面容器与桥接核心）
Wails v3将前端应用封装为原生桌面应用，并提供Go与JavaScript之间的双向通信能力。`main.go`中通过`application.NewService`注册后端服务，实现方法暴露与事件监听。

## 构建与开发流程

### Taskfile.yml（多平台构建协调）
`Taskfile.yml`作为跨平台构建协调文件，通过`includes`引入各平台特定构建配置（Windows、macOS、Linux）。定义统一任务接口：
- `dev`：启动开发模式，调用`wails3 dev`
- `build`：构建应用，按操作系统路由到对应子任务
- `package`：打包生产版本
- `run`：运行应用

该设计实现构建逻辑的平台解耦，提升多环境一致性。

### wails3 CLI工具链
wails3 CLI提供核心开发命令：
- `wails3 dev`：启动开发服务器，支持前后端热重载
- `wails3 build`：生成生产级可执行文件
工具链自动处理前端构建、资源嵌入与二进制打包，简化发布流程。

## 技术选型优势分析

| 技术维度 | 优势说明 |
|---------|---------|
| **开发效率** | TypeScript + Vite + Hot Reload 实现快速迭代，Zustand减少状态管理复杂度 |
| **运行性能** | Go语言高并发处理AI请求，SQLite轻量本地存储，React 19优化渲染性能 |
| **可维护性** | 清晰的分层架构（models/service/storage），类型安全与模块化设计 |
| **跨平台能力** | Wails + Taskfile 支持Windows/macOS/Linux一键构建 |
| **生态兼容性** | Ant Design丰富UI组件，GORM成熟ORM支持，Vite广泛插件生态 |

该技术组合在保证高性能的同时，显著降低开发与维护成本，适合长期演进的桌面AI应用。
