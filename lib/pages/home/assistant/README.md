# 多标签页对话界面

## 功能概述

新的多标签页对话界面 (`MultiTabAssistant`) 为用户提供了更好的多对话管理体验，允许同时打开多个对话并在它们之间快速切换。

## 主要特性

### 1. 多标签页支持
- 支持同时打开多个对话标签页
- 每个标签页独立管理一个对话
- 最多支持5个标签页（可配置）

### 2. 标签页管理
- **新建标签页**: 点击标签栏右侧的 `+` 按钮
- **关闭标签页**: 点击标签页标题右侧的 `×` 按钮
- **切换标签页**: 点击标签页标题或使用键盘快捷键
- **自动保留**: 至少保留一个标签页

### 3. 智能初始化
- 应用启动时自动加载最近的对话作为标签页
- 如果没有历史对话，自动创建欢迎对话
- 支持离线模式和错误恢复

### 4. 美观的UI设计
- **圆角设计**: 标签页和内容区域采用圆角矩形设计
- **现代化样式**: 使用Material Design 3色彩系统
- **视觉层次**: 活跃标签页有明显的视觉反馈
- **阴影效果**: 内容区域具有轻微阴影，增强层次感
- **标题优化**: 显示对话标题，超长时自动截断，最大宽度200px

## 技术实现

### 核心组件

#### MultiTabAssistant
主要的多标签页容器组件，负责：
- 标签页的创建、删除和切换
- TabController的管理
- 与现有AssistantPage的集成

#### AssistantTab
标签页数据模型，包含：
- `id`: 唯一标识符
- `title`: 显示标题
- `conversationManager`: 对话管理器实例

### 架构设计

```
MultiTabAssistant
├── TabBar (标签栏)
│   ├── Tab 1 (对话1)
│   ├── Tab 2 (对话2)
│   ├── ...
│   └── + 按钮 (新建标签页)
└── TabBarView (内容区域)
    ├── AssistantPage (对话1界面)
    ├── AssistantPage (对话2界面)
    └── ...
```

### 与现有系统的集成

1. **替换原有AssistantPage**: 在 `home.dart` 中将 `AssistantPage` 替换为 `MultiTabAssistant`
2. **保持API兼容**: 每个标签页仍使用原有的 `AssistantPage` 组件
3. **独立的ConversationManager**: 每个标签页拥有独立的对话管理器

## 使用方法

### 基本操作

1. **创建新标签页**:
   ```dart
   // 点击 + 按钮或调用
   await _createNewTab('自定义标题');
   ```

2. **关闭标签页**:
   ```dart
   // 点击 × 按钮或调用
   _closeTab(index);
   ```

3. **切换标签页**:
   ```dart
   // 通过TabController切换
   _tabController.animateTo(index);
   ```

### 自定义配置

可以通过修改以下常量来调整行为：

```dart
// 最大标签页数量
const int MAX_TABS = 5;

// 标签页标题最大宽度
const double MAX_TAB_WIDTH = 200.0;

// 是否允许关闭最后一个标签页
const bool ALLOW_CLOSE_LAST_TAB = false;
```

## 注意事项

1. **性能考虑**: 每个标签页都会保持活跃状态，建议限制标签页数量
2. **内存管理**: TabController和ConversationManager会在组件销毁时自动清理
3. **数据同步**: 每个标签页的数据独立，不会相互影响
4. **错误处理**: 包含完整的错误处理和降级策略
