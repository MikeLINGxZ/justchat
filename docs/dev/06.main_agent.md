# 主 Agent 实现

## 概述

基于 [trpc-group/trpc-agent-go](https://github.com/trpc-group/trpc-agent-go) 框架实现 Lemontea 的对话引擎核心功能。该模块负责 AI 对话流程、工具调用、会话管理和任务编排，是整个应用的核心决策和执行引擎。

## 目标

本阶段的目标是实现基础的对话功能和会话管理，为后续的记忆系统、工具调用等高级功能奠定基础。

未来可能的扩展方向：

- 智能化记忆系统
- MCP 集成
- Skills 与 Prompt 管理
- 工作流编排与多 Agent 协作

## 第一阶段：基础对话与会话管理

#### 1.1 功能需求

- **流式对话**

  - 支持向模型发送消息并接收流式响应
  - 支持手动停止生成
  - 支持实时显示回复内容和生成进度
  - 支持工具调用
- **会话管理**

  - 创建新会话
  - 加载会话历史（分页加载）
  - 查询会话列表（分页加载）
  - 重命名会话
  - 删除会话
  - 收藏/取消收藏会话
  - 自动生成会话标题
- **消息管理**

  - 存储用户消息和 AI 回复
  - 支持工具调用信息和结果的存储
- **后台运行与状态反馈**

  - 支持对话在后台运行
  - 后台运行时，对话列表中对应会话标题左侧显示 loading
  - 后台运行因错误停止时，列表标题左侧显示红色点
  - 后台运行因等待用户确认停止时，列表标题左侧显示蓝色点
  - 正常完成时，列表标题左侧显示绿色点
- **用户交互模式**

  - 支持“询问确认”交互：让用户选择确认、拒绝或直接输入内容告诉大模型想法
  - 支持“直接输入”交互：让用户直接输入回答

#### 1.2 数据存储需求

存储对话中所有输入和输出内容，包括思考、工具调用及其结果、用户输入等

#### 目录结构

```
backend/
  pkg/
    agent/
      manager.go              # Agent 生命周期管理
      chat_handler.go         # 聊天处理逻辑
      stream_manager.go       # 流式输出管理
  service/
    agent/                    # Agent 服务（Wails 绑定）
  storage/
    session.go                # 会话存储
    message.go                # 消息存储
```

### 模块通信

```
Wails Frontend
      ↓
Wails Service (Agent)
      ↓
pkg/agent/{manager/chat_handler/stream_manager}
      ↓
provider (Models)
      ↓
{OpenAI/Claude/LocalModel}
```
