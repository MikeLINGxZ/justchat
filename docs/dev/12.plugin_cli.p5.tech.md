# CLI 插件实施计划 — P5 程序内交互式登录

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让用户无需打开终端就能在应用内完成 CLI 插件的交互式登录（如飞书 `lark-cli config init --new` 的扫码授权流程）。后端用 PTY 跑 `manifest.executable + manifest.login_command`，前端用 xterm.js 渲染输出并把键盘事件回传到子进程的 stdin。

**Architecture:** `pkg/cli` 新增 `LoginSession`，封装 PTY master/slave 与子进程生命周期；`Manager.StartLogin` 加载 manifest、注入 isolated env、启动会话。`service/plugin` 维护按扩展 ID 唯一的活跃会话表，新增三个 Wails 方法（`LoginCli` / `SendLoginStdin` / `CancelLoginCli`）和两个事件（`cli.login.output` / `cli.login.done`）。前端在详情页加「登录 / 认证」按钮和「使用系统全局登录态」开关，按钮触发后弹出 xterm.js 终端模态框，订阅事件并把 stdin 转发回去。

**Tech Stack:** Go + `github.com/creack/pty` + Wails v3 + React + xterm.js + Tailwind。

参考需求文档：`docs/dev/12.plugin_cli.md` §6 登录态管理、§7.1 详情面板、§9 错误处理。所有实现遵循 `docs/dev/00.rules.md`。

**本期实现：**

### 接口

```go
// LoginSessionParams 描述一次登录会话的输入。
type LoginSessionParams struct {
	Executable string   // 绝对路径，来自 manifest.Executable
	Argv       []string // manifest.LoginCommand（不含 Executable）
	Env        []string // 由 BuildEnv 生成的 isolated/shared env
	Cwd        string   // 通常是 plugins/cli/<name>，可空
}

// LoginSession 持有一个 PTY 子进程的全生命周期。所有公共方法都是 goroutine-safe。
type LoginSession struct {
	// 私有字段：cmd *exec.Cmd, ptmx *os.File, output chan []byte, done chan struct{},
	// waitErr error, exitCode int, mu sync.Mutex, closed bool
}

// StartLoginSession 立刻 fork+exec 登录命令。返回后 Output() 已经在写入。
func StartLoginSession(ctx context.Context, p LoginSessionParams) (*LoginSession, error)

// Output 返回只读输出通道。子进程退出时通道关闭。每个 chunk 是 PTY master 的一次读取结果。
func (s *LoginSession) Output() <-chan []byte

// Write 把数据原样写入 PTY master（也就是子进程的 stdin）。回车 / Ctrl-C 由调用方按 xterm 协议自行编码。
func (s *LoginSession) Write(data []byte) error

// Resize 同步终端窗口尺寸（让 CLI 输出按窗口换行）。前端 resize 时调一次。
func (s *LoginSession) Resize(rows, cols uint16) error

// Cancel 先 SIGTERM、1.5s 内未退则 SIGKILL。已退出时返回 nil。
func (s *LoginSession) Cancel() error

// Wait 阻塞直到子进程退出，返回 exit code 与错误。已退出时立刻返回。
func (s *LoginSession) Wait() (exitCode int, err error)
```

### 实现要点

1. `StartLoginSession` 内部用 `exec.CommandContext(ctx, p.Executable, p.Argv...)`，设 `cmd.Env = p.Env`、`cmd.Dir = p.Cwd`。
2. `ptmx, err := pty.Start(cmd)` 拿到 master。失败时回收资源后返回错误。
3. 起一个 goroutine 循环 `ptmx.Read(buf)`（buf 4 KiB），把 buf[:n] copy 一份送进 `output` chan。EOF / 错误 → 关闭 chan、调 `cmd.Wait()` 拿 exit code、关闭 `ptmx`、关闭 `done`、置 `closed=true`。
4. `Write` / `Resize` / `Cancel` 在 `closed==true` 时直接返回 `errors.New("cli: login session already closed")`。
5. `Cancel` 实现：`cmd.Process.Signal(syscall.SIGTERM)`；起 timer 1.5s，到点若 `<-done` 仍未 ready，则 `cmd.Process.Kill()`。
6. **严禁** 在退出路径上 close 已经 close 的 chan / fd —— 用 `sync.Once` 包一下清理逻辑。

### 测试

测试用 `t.Skip` 在非 Unix 跳过（`runtime.GOOS == "windows"`）。

1. `TestLoginSessionEchoesStdin`：写一个 bash bin，内容 `read line; echo "got: $line"; exit 0`。Start → Write("hello\n") → 从 Output() 读到包含 `got: hello` 的 chunk → Wait() 拿 exit 0。
2. `TestLoginSessionCancelTerminatesChild`：bash bin `sleep 30`。Start → Cancel() → Wait() 在 2s 内返回，exit code 非 0 表示被信号杀掉。
3. `TestLoginSessionWriteAfterCloseErrors`：跑一个立刻 exit 的 bin，Wait()，然后 Write → 期望 `already closed` 错误。
4. `TestLoginSessionResize`：仅断言 Resize 不报错（PTY 接收 TIOCSWINSZ 没有可观察输出）。

### Wails 方法（plugin_login.go）

```go
// LoginCli starts an interactive login session for the given CLI extension.
// Stdout/stderr stream to the `cli.login.output` event; completion fires `cli.login.done`.
// Returns ErrCliLoginSessionConflict when another session is already active for this extension.
func (p *Plugin) LoginCli(ctx context.Context, input plugin_dto.LoginCliInput) (*plugin_dto.LoginCliOutput, error)

// SendLoginStdin forwards a chunk of user input to the active session.
func (p *Plugin) SendLoginStdin(ctx context.Context, input plugin_dto.SendLoginStdinInput) error

// ResizeLoginCli updates the PTY window size for an active session.
func (p *Plugin) ResizeLoginCli(ctx context.Context, input plugin_dto.ResizeLoginCliInput) error

// CancelLoginCli sends SIGTERM (then SIGKILL after 1.5s) to the active session.
func (p *Plugin) CancelLoginCli(ctx context.Context, input plugin_dto.CancelLoginCliInput) error
```

### 实现要点

`LoginCli` 主体：

1. 查 `loadAgentConfig()` 找到 `ExtensionItem` by ID；要求 `kind == "cli"` 且 `Enabled == true` 且 `RootDir` 存在 —— 否则 `ErrCliLoginNotFound`。
2. 解析 `ExtensionItem.Name` 出插件名（与 `RunTool` 共用 helper —— 如果还没抽取就抽一个 `pluginName(item)`）。
3. `p.loginMu.Lock()` → 检查 `loginSessions[item.ID]` 是否非 nil → 是则解锁返回 `ErrCliLoginSessionConflict`。
4. 调 `p.cliManager.StartLogin(ctx, name)`，成功后写入 map，解锁。
5. 起 goroutine：
   ```go
   defer func() {
       p.loginMu.Lock()
       delete(p.loginSessions, item.ID)
       p.loginMu.Unlock()
   }()
   for chunk := range session.Output() {
       p.wailsApp.Event.Emit("cli.login.output", loginOutputPayload{
           ID:   item.ID,
           Data: string(chunk),
       })
   }
   code, waitErr := session.Wait()
   p.wailsApp.Event.Emit("cli.login.done", loginDonePayload{
       ID:       item.ID,
       ExitCode: code,
       Error:    safeErr(waitErr),
   })
   ```
6. 立刻返回 `(&plugin_dto.LoginCliOutput{}, nil)`，输出走事件。

`SendLoginStdin`：取 session，session 为 nil 返回 `ErrCliLoginNotFound`，否则 `session.Write([]byte(input.Data))`。

`ResizeLoginCli`：同上，`session.Resize(input.Rows, input.Cols)`。

`CancelLoginCli`：同上，`session.Cancel()`；不删 map（让 Output goroutine 在 Wait 后自然清理）。

### 组件接口

```tsx
// CliLoginDialogProps controls a modal terminal that streams a backend PTY login session.
export type CliLoginDialogProps = {
  open: boolean
  extensionId: string
  title: string
  onClose: () => void              // user dismissed the modal — caller decides whether to also cancel
  onCancel: () => void             // explicit Cancel button — calls CancelLoginCli + onClose
  onDone?: (exitCode: number) => void
}

export function CliLoginDialog(props: CliLoginDialogProps): JSX.Element | null
```

### 行为

1. `useEffect(open)`：mount 时 `new Terminal({ convertEol: true, fontFamily: 'Menlo,Consolas,monospace', fontSize: 13, theme: { background: '#1b2636' } })`，挂载到 ref 容器；attach `FitAddon` 并 fit。
2. 订阅 `Events.On('cli.login.output', ...)`：收到 payload.ID === extensionId 时 `terminal.write(payload.data)`。
3. 订阅 `Events.On('cli.login.done', ...)`：payload.ID 匹配时调 `onDone(payload.exit_code)` 并显示完成横幅（成功绿、失败红，倒计时 3s 自动关闭或留按钮）。
4. `terminal.onData((data) => PluginBinding.SendLoginStdin(new SendLoginStdinInput({ id: extensionId, data })))` —— xterm 已经做了按键 → ANSI 序列编码（含回车 `\r`），直接透传给 PTY。
5. `ResizeObserver` 监听容器大小，每次回调 fit() 完后调 `PluginBinding.ResizeLoginCli({ id, rows, cols })`。
6. 卸载（用户关弹窗）时清理：取消事件订阅、`terminal.dispose()`。**不**自动调 CancelLoginCli —— 关闭弹窗与杀进程是两个语义，由父组件决定。

## 难点与未决项

1. **PTY 跨平台**：creack/pty 在 Windows 上仅做最小骨架，二维码渲染等可能错位。本期跳过 Windows。
2. **xterm + Wails 事件吞吐**：`Events.On` 回调里直接 `terminal.write()`，PTY 高速输出（如长 ANSI 二维码刷新）可能引发 React 渲染抖动 —— 实测若卡顿就用 `requestAnimationFrame` 合批。
3. **隔离 ↔ shared 切换的副作用**：切到 shared 后下次工具调用会读宿主 HOME，如果宿主未登录会突然出现「未登录」错误。这是预期行为，但需要在文案上让用户感知（已在 hint 里说明）。
4. **会话清理时机**：用户关闭设置窗口（不仅关弹窗）时也应触发 CancelLoginCli。这是 SettingsApp 的 `useEffect` 清理职责，留意 P5 实施时验证：关 settings 窗口 → `ps aux` 看不到孤儿 lark-cli 进程。
5. **AI 触发登录（§6.2）延后**：合并到 P6（与 meta tool install_cli_plugin 一起），因为同样要在聊天里走审批气泡 + 调用 PluginBinding.LoginCli 触发弹窗 —— 复用本期产出的全部基础设施。
