# 错误消息需求

## 背景

后端所有 Wails 绑定方法（`backend/service/*`）均返回 `ierror.IError` 类型错误，包含：
- `Msg`：本地化用户友好提示（如"发送消息失败"）
- `Detail`：原始错误字符串（底层 Go error.Error() 内容）

前端需要捕获并展示这些错误。

---

## 错误展示规则

### 展示形式：聊天内错误气泡

所有来自 backend/service 的错误，均以**错误消息气泡**的形式插入到聊天记录中：

1. 气泡左侧显示本地化的 `Msg` 内容
2. 气泡右侧有一个感叹号图标
3. 点击感叹号图标向下展开，显示 `Detail`（原始错误字符串）

### 触发场景

| 场景 | 说明 |
| ---- | ---- |
| 流式 LLM 报错 | 大模型在流式输出过程中返回错误（如 API 超限、网络中断） |
| 发送消息前校验失败 | 如附件数量超限、附件路径无效等 |

### 已知问题（同步修复）

**流式 LLM 报错后，助手消息气泡显示空白**：
- 根因：后端发出 `agent:stream:error` 事件后会将错误写入 DB，但前端事件处理器未重新加载消息列表
- 修复：stream error handler 补充 `loadMessages` 调用

---

## 技术实现要点

### 后端

1. **`ierror.IError` 结构体**
   - 将 `Err error` 字段改为 `Detail string`（原因：`error` 接口序列化为 JSON 时丢失内容）
   - `Error(code, err)` 构造器中 `Detail: err.Error()`

2. **流式错误消息持久化**（`chat_handler.go`）
   - 存储错误消息时使用 `ContentType: "error"`（区别于普通 `"text"`）
   - `Content` 字段存储 JSON：`{"msg": "<本地化消息>", "detail": "<原始错误>"}`
   - 新增 i18n key `ierror.agent.stream_error`（中英文各一条）

3. **i18n 新增**
   - `ierror.agent.stream_error` = "大模型响应出错" / "Model response error"

### 前端

1. **`types/index.ts`**：`MessageContentType` 新增 `"error"`

2. **`chatStore.ts`**
   - `agent:stream:error` 事件处理器新增 `void get().loadMessages(data.sessionId)`
   - `sendMessage` 加 try/catch，catch 时将错误解析为 `contentType: "error"` 的临时消息插入本地 messages

3. **新组件 `ErrorMessageBubble.tsx`**（`components/chat/`）
   - Props：`content: string`（JSON 字符串）
   - 渲染：msg 文本 + 感叹号图标，点击展开 detail
   - 遵循双主题（light/dark）和字体大小规范

4. **`MessageItem.tsx`**：新增 `contentType === 'error'` 分支，渲染 `ErrorMessageBubble`

5. **i18n（前端）**：新增 `chat.errorDetail` / `chat.hideErrorDetail` 等翻译 key

---

## 不在本次范围内

- 其他 Wails binding 错误（如删除会话、重命名等操作失败）暂不处理
- Toast 通知组件（本次统一用聊天气泡）
