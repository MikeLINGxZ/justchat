# 插件与工具

我正在基于 `wails3` 开发一个 AI Chatbox，目前基础功能已完成。接下来希望为这个 Chatbox 增加插件和工具能力：

* **插件** ：类似 VS Code 的插件模型，基于 `Node.js` 开发，能够访问 Chatbox 内部提供的各类 API，并支持提供自定义视图能力。
* **工具** ：基于 MCP 的工具体系。

我们计划同时支持 MCP 工具和插件机制，本期重点先实现对 MCP 工具的支持。

## 前端界面

### 页面结构

* 左侧是二级菜单/列表区域，右侧是详情区域。
* 整体样式与 `模型供应商` 页面保持一致。

### 左侧列表

* 顶部显示标题：`插件与工具`
* 标题右侧有一个“添加”按钮，点击后展开菜单：
  * `mcp工具`
  * `插件`
* 选择后弹出文件夹选择器，用户选择对应的插件/工具目录。

### 列表项显示内容

每个已添加的工具/插件列表项展示：

* 名称
* 描述
* 作者
* 版本
* 类别标签：`mcp工具` 或 `插件`
* 启用 / 禁用开关

右侧还有“更多”按钮，点击展开菜单：

* 重新加载
* 删除

### 右侧详情面板

点击左侧某个工具/插件后，右侧显示该项详情，包括：

* 名称
* 版本
* 作者
* 描述
* 启用 / 禁用开关
* 配置文件（若为 MCP 工具，则显示其配置文件信息，配置文件可以修改，配置文件区域显示恢复和应用按钮，按钮默认禁用，当有修改的时候，按钮启用）

### 体验说明

* 添加成功后，该工具/插件应立即出现在列表中。
* 列表与详情保持联动：选择左侧项目时右侧自动刷新对应详情。
* “更多”菜单提供后续管理操作，不影响当前选中的详情展示。

## 操作逻辑

* 用户新增工具后，输入框中的工具列表要同步更新。
* 已启用的 `mcp` 工具应出现在输入框可选工具中，供大模型直接调用。
* 工具列表底部固定显示一个“管理工具与插件”按钮，点击即可打开设置页的“插件与工具”界面。

## 后端逻辑

* 用户添加 `mcp` 工具后，需将该工具复制到数据目录下的 `mcp` 子目录。
* 存放路径为：`{data_dir}/mcp/{mcp_name}/{version}`。
* 若该工具没有版本号，则直接存放到：`{data_dir}/mcp/{mcp_name}/default`。
